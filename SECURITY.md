# 安全策略 (Security Policy)

我们要感谢您对 **工业流程 ERP 系统 (开源版)** 安全性的关注。我们非常重视系统的安全性，并致力于保护用户的数据安全。

## 📦 支持的版本 (Supported Versions)

为了确保安全，请始终使用最新版本的代码。目前我们主要维护以下版本：

| 版本 | 支持状态 |
| :--- | :--- |
| `main` 分支 | ✅ 接受安全更新 |
| 历史 Release | ❌ 不再维护 |

## 🐞 报告漏洞 (Reporting a Vulnerability)

如果您在本项目中发现了安全漏洞，**请不要在 GitHub Issues 中公开披露**。公开漏洞可能会被恶意利用，从而威胁到其他用户的安全。

请通过以下方式直接联系维护者：

- **联系人**: fortitudelucifer
- **反馈渠道**: 请发送邮件至 (您可以在 GitHub 个人主页找到我的联系方式) 或通过 GitHub 的私信功能（如果可用）。
- **内容建议**:
    - 漏洞的类型（例如 SQL 注入、XSS、越权访问等）。
    - 受影响的代码文件或模块。
    - 复现漏洞的步骤或 Proof of Concept (PoC) 代码。
    - 您认为的潜在影响。

## 🛡️ 响应流程

1.  **确认**: 我会在收到报告后尽快确认收到，并开始评估漏洞的严重程度。
2.  **修复**: 一旦确认漏洞存在，我们会尽快开发修复补丁。在此期间，请您协助保密。
3.  **发布**: 修复完成后，我们会合并代码并发布新的安全更新。
4.  **致谢**: 在确认修复且发布后，如果在此之前漏洞未被公开，我们会在更新日志或致谢名单中公开感谢您的贡献（如果您愿意）。

## ⚠️ 常见安全提示

由于本项目是一个 ERP 系统，涉及企业敏感数据，建议在部署时遵循以下安全最佳实践：

1.  **修改默认密钥**: 务必在 `config.py` 中将 `SECRET_KEY` 修改为一个复杂的随机字符串。
2.  **内网部署**: 强烈建议将数据库和应用部署在内网环境，通过 VPN 或 Tailscale 等安全隧道访问，避免直接暴露在公网。
3.  **数据库权限**: 使用最小权限原则配置数据库账号，不要使用 `sa` 或 `root` 账号运行应用。
4.  **HTTPS**: 如果必须通过公网访问，请务必配置 HTTPS。
